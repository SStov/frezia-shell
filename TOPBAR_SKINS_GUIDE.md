# Отчёт по рефакторингу TopBar и руководство по созданию скинов

## 1. Отчёт о проведённой работе

В рамках задачи по переводу панели (TopBar) на модульную архитектуру было сделано следующее:

1. **Декомпозиция монолита**:
   - Весь код из огромного `TopBar.qml` был разделён на независимые компоненты.
   - Каждый виджет теперь живёт в своём отдельном файле в папке `widgets/topbar/` (например, `ClockWidget.qml`, `WorkspacesWidget.qml`, `PowerButton.qml` и т.д.).
   - Создан файл `qmldir`, который регистрирует все эти виджеты, чтобы их можно было легко импортировать в любой скин.

2. **Динамическая загрузка стилей**:
   - Сам `TopBar.qml` теперь выполняет роль "хоста" — он только создаёт подложку и содержит внутри себя динамический `Loader`, который подтягивает файл разметки (скина).
   - В меню настроек (`SettingsMenu.qml`) добавлен выбор стиля (Стиль панели: `Default` или `Alternative`). Настройка сохраняется в `settings.qml` в свойство `topBarStyle`.
   
3. **Исправление багов инициализации (Жизненный цикл QML)**:
   - Исправлена классическая ошибка невидимых контейнеров: корневым элементам скинов было явно задано свойство `anchors.fill: parent`.
   - Решена проблема с порядком загрузки, когда данные слотов (`topBarOrder`) пытались загрузиться до того, как переменная `bar` была инициализирована.
   - Загрузка виджетов в слоты `Default.qml` переведена на **чисто декларативный** реактивный подход (использование `sourceComponent` с тернарными операторами).

4. **Фикс путей импорта**:
   - В связи с тем, что виджеты переехали на один уровень глубже (в папку `topbar`), пути импорта для глобальных модулей были изменены на `import "../../core"` и `import "../../components"`.

---

## 2. Инструкция: Как создать и добавить новый скин (стиль) для панели

Чтобы создать новый, альтернативный внешний вид для панели (например, `MacOSStyle.qml`), выполните следующие шаги:

### Шаг 1: Создание файла разметки
В папке `widgets/topbar/` создайте новый файл. Имя файла должно начинаться с большой буквы. Например, `MacOSStyle.qml`.

### Шаг 2: Базовая структура файла
Любой скин **обязан** иметь определённую базовую структуру, чтобы корректно отображаться и получать данные от основного бара:

```qml
import QtQuick
import QtQuick.Layouts
import "."                 // Импорт локальных виджетов из qmldir
import "../../core"        // Импорт Colors, Settings и т.д.
import "../../components"  // Общие компоненты (StyledText и т.д.)

Item {
    id: rootLayout
    
    // ОБЯЗАТЕЛЬНО: свойство bar для получения контекста панели
    property var bar
    
    // ОБЯЗАТЕЛЬНО: растянуть корневой элемент на всю панель, иначе будет 0x0
    anchors.fill: parent

    // === ВАША РАЗМЕТКА ЗДЕСЬ ===
    RowLayout {
        anchors.fill: parent
        anchors.margins: 10
        spacing: 15
        
        // Пример добавления готового виджета (обязательно передаем bar)
        WorkspacesWidget { bar: rootLayout.bar }
        
        // Пружина, чтобы растолкать виджеты по краям
        Item { Layout.fillWidth: true }
        
        ClockWidget { bar: rootLayout.bar }
        SystemTrayWidget { bar: rootLayout.bar }
    }
}
```

> **Важно**: Каждому виджету (например, `ClockWidget`, `TamagotchiButton`) обязательно нужно пробрасывать свойство `bar: rootLayout.bar`, чтобы виджет мог взаимодействовать с ядром (например, открывать попапы, узнавать текущую раскладку и т.д.).

### Шаг 3: Динамические слоты (Опционально)
Если вы хотите, чтобы ваш скин поддерживал сортировку виджетов из меню настроек (как это делает `Default.qml`), используйте встроенные `Loader`. 
*Пример реализации есть в файле `Default.qml` — там виджеты объединяются в массив `componentsList`, а `Loader` реактивно выбирает нужный по индексу из `rootLayout.bar.topBarOrder`.*

### Шаг 4: Регистрация нового скина в qmldir
Откройте файл `widgets/topbar/qmldir`. 
Добавьте в самый низ новую строку регистрации:
```
MacOSStyle 1.0 MacOSStyle.qml
```
Это позволит QML-движку видеть ваш новый скин.

### Шаг 5: Добавление опции в меню настроек
Чтобы скин можно было выбрать в интерфейсе:
1. Откройте `widgets/SettingsMenu.qml`.
2. Найдите блок настроек панели (поиск по `"Стиль панели"`).
3. Добавьте ваш скин в массив `options` у компонента `OptionSelector`:
```qml
OptionSelector {
    title: "Стиль панели"
    options: ["Default", "Alternative", "MacOSStyle"] // Добавили новый вариант
    currentValue: sysSettings.topBarStyle
    onValueChanged: (val) => {
        sysSettings.topBarStyle = val
        settings.saveSettings(sysSettings)
    }
}
```

### Шаг 6: Готово!
Теперь, если в меню настроек выбрать `MacOSStyle`, файл `TopBar.qml` автоматически подгрузит `topbar/MacOSStyle.qml`, передаст ему `rootBar`, и ваш новый визуальный стиль моментально применится к панели!
